## About

[Gitmoji](https://gitmoji.surge.sh) is an initiative to standardize and explain **the use of emojis on GitHub commit messages**.

**Using emojis** on **commit messages** provides an **easy way** of **identifying the purpose or intention of a commit** with only looking at the emojis used. As there are a lot of different emojis I found the need of creating a guide that can help to use emojis easier.

## Using [gitmoji-cli](https://github.com/biezhi/gitmoji-cli)

To use gitmojis from your command line install [gitmoji-cli](https://github.com/biezhi/gitmoji-cli). A gitmoji interactive client for using emojis on commit messages.

```bash
npm i -g gitmoji-cli
```

## Contributing to gitmoji

Contributing to gitmoji it's a piece of :cake:, read the [contributing guidelines](https://github.com/biezhi/gitmoji/blob/master/.github/CONTRIBUTING.md). You can discuss emojis using the [issues section](https://github.com/biezhi/gitmoji/issues/new). To add a new emoji to the list create an issue and send a pull request, see [how to send a pull request and add a gitmoji](https://github.com/biezhi/gitmoji/blob/master/.github/CONTRIBUTING.md#how-to-add-a-gitmoji).
